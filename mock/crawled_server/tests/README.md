# StaticLand Test Suite

このディレクトリにはStaticLandアプリケーションのテスト関連ファイルが含まれています。

## ディレクトリ構成

```
tests/
├── README.md                 # このファイル
├── test-staticland.js       # Playwrightブラウザテスト
├── screenshots/             # スクリーンショット出力（Git除外）
├── logs/                    # サーバーログ（Git除外）
└── reports/                 # テストレポート（Git除外）
```

## テスト実行方法

### 1. UIテスト（Playwright）

```bash
# サーバーを別ターミナルで起動
npm run dev

# UIテストを実行
npm run test:ui
```

### 2. 開発サーバー（ログ付き）

```bash
# サーバーをログファイル付きで起動
npm run dev:log
```

### 3. テスト成果物のクリーンアップ

```bash
# スクリーンショット、ログ、レポートを削除
npm run test:clean
```

## テスト内容

### 📡 接続性テスト
- サーバーレスポンス確認
- ネットワーク待機処理

### 🏠 ホームページテスト
- タイトル表示確認
- 記事カード表示数チェック
- ナビゲーション要素検証

### 📄 記事詳細ページテスト
- 記事タイトルと構造確認
- 目次リンク動作
- 関連記事表示

### 🏷️ タグページテスト
- タグフィルタリング機能
- タグナビゲーション

### 📱 レスポンシブテスト
- モバイル・デスクトップ表示切り替え
- ナビゲーション適応確認

### ⚡ パフォーマンステスト
- ページ読み込み時間測定
- 初回描画時間計測

### 📸 ビジュアルテスト
- フルページスクリーンショット撮影
- ビジュアル回帰テスト用

## Git管理について

以下のファイル・ディレクトリはGitで追跡されません：

- `tests/screenshots/` - スクリーンショット出力
- `tests/logs/` - サーバーログファイル
- `tests/reports/` - テストレポート
- `*.log` - 全ログファイル
- `*.png`, `*.jpg` など - 画像ファイル

これらは`.gitignore`で除外設定済みです。

## 注意事項

- UIテスト実行前にサーバーが起動していることを確認してください
- テスト実行後は`npm run test:clean`でファイルをクリーンアップできます
- スクリーンショットは`tests/screenshots/`に保存されます
